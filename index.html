Apresentação do curso

Voltar ao topo

Seja bem vindo ao curso de Introdução ao HTML na prática!

O ojetivo geral desse curso é que o estudante aprenda a instalar e configurar o ambiente de desenvolvimento, entender o funcionamento do inspetor de elementos do navegador e criar sua primeira página HTML.

Isso significa que, ao final desse curso, o estudante será capaz de criar seu próprio site! De forma gradual, serão apresentadas ferramentas e conceitos que auxiliem na criação do conjunto de páginas que irá caracterizar o site. Então, sem mais delongas, vamos dar início ao curso!

Download do VSC

Antes de começar a codar, precisamos de um ambiente adequado para isso. Desse modo, o primeiro passo do curso será o donwload do software Visual Studio Code (VSC). O VSC é uma ferramenta de código aberto, o que significa que ele é de graça. Poderiam ser utilizadas outras ferramenas, entretando, para este curso, foi escolhido o VSC. Inspetor de Elementos

Voltar ao topo

Uma ferramenta muito útil para o desenvolvedor é o Inspetor de Elementos. Essa é uma ferramenta disponibilizada por navegadores, como Google Chrome, Firefox, Ópera, etc. Novamente, para este curso utilizaremos o Google Chrome.

Para acessar a ferramenta é necessário estar em um site, pressionar o F12 no teclado, o que irá abrir o console. Alternativamente, você pode clicar nos três pontos, situados no canto superior direito, em seguida em "mais ferramentas" e por fim "Ferramentas do desenvolvedor". Com o console aberto, é possível verificar algumas caracerísticas que foram utilizadas na página.

No canto superior esquerdo do console está o ícone de inspeção de elementos. Ao clicar nele, você poderá clicar em um elemento da página para verificar as suas características e como ele foi formatado (em CSS). Também funciona ao se clicar em uma das linhas do console, que irá destacar o elemento na página o qual a linha se refere.

Além de verificar as características que aparecem no novo console do lado direito, você pode alterar essas características, como a cor do elemento. Importante ressaltar que isso não altera o site em sim, e sim a sua vizualização. Essa informação também é válida para o console HTML, nos quais as linhas possuem os elementos programados.

Essa ferramenta auxilia no estudo em HTML, além de apresentar ao vizualizador opções de comandos que possam ser utilizados em futuros projetos.

Estrutura Básica do HTML

Neste tópico iremos tratar da estrutura básica de um arquivo HTML. Para iniciarmos, abra o Visual Studio Code e defina a pasta onde será salvo o projeto clicando em "Arquivo", "Abrir Pasta" e selecione a de sua preferência.

Após a seleção da pasta, na área do Explorador, o VSC irá mostrar para você no lado esquerdo toda a estrutura que está sendo desenvolvida para as suas páginas. Observação

Para observar a página no navegador pelo Live Server, o usuário deverá ir até o navegador e clicar com o botão direito no arquivo gerado (como, por exemplo, o arquivo index.html), selecionar a opção "open with Live Server. Para observar as alterações, o programador deve salvar as alterações no VSC pressionando "Crtl+S" ou indo em "Arquivo" e depois pressionar "Salvar".

Tags de estruturação

HTLM

É a parte onde está condida a programação da página, definindo que os comando inseridos pertencem à um arquivo HTML. Iniciando o arquivo com "<html>" e o encerrando em "</html>".

Importante ressaltar que toda a tag que será utilizada em HTML deverá conter os símbolos "<" e ">" e um texto característico entre eles. Com o decorrer do curso ficará mais visível esse padrão. O simbolo "/" presente no início de uma tag denota que o programador está encerrando a área de atuação dela.

A tag "<html>" é dividida em dois seguimentos:

Head

A cabeça, ou cabeçalho irá carregar informações antes de exibir para o usuário e podem ser utilizadas outras tags para configuração como a tag title.

Title

A tag "Title" é utilizada dentro do da tag "head"" para dar o título da página.

Body

Já o body, ou corpo, é onde estará contido o que será apresentado para o usuário na página.

Essas três tags compoêm a parte básica estrutural do arquivo HTLM. No próximo tópico, veremos o que são essas tão mencionadas tags e como elas auxiliam e modificam a página.

Falando sobre Tags

Voltar ao topo

Tags são elementos que inserem e delimitam funções que tem como objetivo caracterizar e modificar como o seu arquivo HTML irá ser apresentado para o usuário da página. No tópico passado, introduzimos três tags que constituem a estrutura básica de um arquivo HTML, as tags <html>, <head> e <body>.

Existem milhares de tipos de tags, como por exemplo as tags de alteração de texto <i>, <strong> e <u>, que tem as funções de deixar o texto em itálico, negrito e sublinhado. Elas funcionam da seguinte forma, para deixar o texto com a característica escolhida, "abra" a tag, por exemplo <strong>, digite o texto e feche a tag, </strong>. Caso você esteja usando o VSC, ao abrir uma tag ele já irá sugerir o seu fechamento ao lado, o que pode lhe poupar tempo.

Para combinar tags, basta abrir uma tag, abrir a outra, digitar o texto, e fechar na ordem oposta de abertua, se você começou com a tag <strong> e depois inseriu a tag <i>, deve fechar primeiro a última tag inserida e caminhando de trás para frente até o fechamento da primeira tag. Lembrando que, se você deseja que apenas parte de um texto seja afetado pelo recurso escolhido, abra a tag imediatamente antes do texto selecionado, e a feche logo após.

Importante ressaltar, existem tags que não precisam de tag fechamento, como a tag <input>, que tem a função de inserir um campo na sua página para interação do usuário. Tags desse tipo se "auto fecham", não necessitando da estutura vista nas tags mencionadas no parágrafo anterior.

Essa é a lógica básica que envolve a utilização de tags. É importante pesquisar o que se deseja e como a tag se comporta antes de utilizá-la. Além disso, no próximo tópico, veremos como é possível modificar o comportamento das tags para nos retornar diferentes respostas, é o que chamamos de "Atributos".

Atributos de Tags

Voltar ao topo

As tags possuem comandos que podem alterar as configurações de seu comportamento básico. Os atributos podem ser os comuns de todas as tags, e os específicos de cada.

Atributos globais

id

Um exemplo de tag global é o "id", que gera uma identificação ao elemento escolhido. Para adicionar uma identificação, basta ir dentro da tag e digitar id="___" (no VSC, as aspas carregam automáticamente), onde no espaço entre as aspas é digitado o texto de identificação. Assim, por exemplo, você poderá identificar no javascript qual elemento se deseja mexer.

style

Outro exemplo de atributo é o style, que tem a função de formatar o estilo do seu texto. Por exemplo, mudar a cor, comando style="color: blue". Esse é um exemplo de comando CSS, que tem justamente a finalidade de formatação da página HTML de modo estético.

class

Além desses, pode-se inserir o atributo "class", que irá categorizar elementos. Sua utilização é similar ao atributo "id", mas, diferentemente do atributo "id", o "class" irá inserir o elemento escolhido em um grupo de elementos. Assim, o desenvolvedor poderá definir características específicas para aquele grupo, como cores e estilo. Atributos que buscam formatar ou personalizar a aparência da página serão melhor detalhados no curso de CSS.

Atributos específicos

Exemplos na tag input

Como visto no tópico Falando sobre Tags, o comando <input> insere um campo digitável na página. Um exemplo de atributo específico para essa tag é "type" que define qual o tipo de entrada pode ser utilizada no campo digitável. Caso você escreva <input type="text">, o campo só receberá entradas de texto, já se você moficiar para <input type="number">, o campo agora só irá aceitar números (ressala: aceita a letra "e" por questões de números exponenciais). Curiosamente, você também pode aplicar esse atributo seguinte froma: <input type="color">, assim o campo interativo será definido por cores que podem ser selecionadas pelo usuário.

Exemplos na tag img

Ao se utilizar a tag img, é necessário acrescentar minimamente um atributo para que ela possa funcionar de maneira correta. O atributo em questão é o src, do inglês source que significa a fonte da imagem, que pode ser um endereço na internet ou no seu próprio computador.

Além da fonte da imagem, muitas vezes será necessário formatar o tamanho dela, e isso pode ser executado através do atrabuto width, no qual você pode definir em pixels o tamanho da imagem.

Esses são alguns exemplos de atributos que podem ser inseridos nas tags, o importante, assim como pesquisar como usar uma tag, é saber quais atributos ela pode suportar. Ao decorrer do curso ao se falar a palavra, atributo, o desenvolvedor em potêncial deve já associar ao que foi explicado nesse tópico.

Textos

Voltar ao topo

Este tópico abordará a tipografia da sua página e as principais tags relacionadas a configuração de textos. Durante o tópico você pode utilizar a ferramenta do desenvolvedor para verificar os pontos apresentados, bem como o espaçamento automático (na cor laranja caso você esteja utilizando o Google Chrome) que o navegador formata automaticamente dependendo do tipo de tag utilizada. Além disso, verificar que a página também irá modificar o tamanho do texto com base na janela apresentada.

Headers

Hearders ou cabeçalhos, servem para criar níveis de título em sua página. Você deve ter reparado que essa página está estruturada em níveis, com tópicos e subtópicos sendo diferenciados por tamanho. Quando maior, mais alto na hierarquia. Para configurar esses cabeçalhos, utilizamos a tag <hx>, onde o "x" deve ser substituído pelo nível a ser designado (de 1 a 6). Logo, <h1> corresponde ao primeiro nível, <h2> ao segundo e assim sucessivamente.

Parágrafos

Já o corpo do texto não é um tipo de cabeçalho, e sim parágrafos. Para designar um texto como tal, utilizamos a tag <p>. A cada parágrafo novo, devemos abrir uma nova tag e fechá-la ao final.

Citações

Além de textos produzidos por você, talvez haja a necessidade de incluir citações de outras pessoas. Para facilitar a formatação desse tipo de texto, é utilizada a tag <blockquote>.

42.

Essa tag desprende o conteúdo dela do restante do texto. Repare que podemos utilizar outras tags, como a <p>, para melhorar a apresentação dessa citação.

"42."
O guia do Mochileiro da Galáxias.

Em CSS podemos formatar ainda melhor o texto, mas é necessário ter um HTML bem estruturado para poder trabalhar melhor com o CSS.

Exemplos de tags de formatação

No tópico Falando sobre Tags vimos <i>, <strong> e <u>, que tem as funções de deixar o texto em itálico, negrito e sublinhado. Além dessas, podemos citar algumas outras: A tag <mark>, que acrescenta um destaque como se fosse um marcador de texto; As tags <sup> e <sub>, que colocam textos como índices superiores e inferiores, respectivamente.

Exemplos: 22é igual a 4. H2O é a formula da água.

Outro exemplo que pode vir a calhar é a utilização da tag <del>, que indica a retirada de uma parte do texto, o que pode ser visto em leis quando sofrem alterações. É uma informação que não está atualizada, mas que não deve ser apagada do texto.

Exemplo:O horário de verão irá começar à meia noite.

Esses são alguns exemplos de tags que podemos utilizar para modificar o texto em nossos arquivos HTML. Para um maior apronfudamento, sugerimos o site da W3 schools, que possui uma lista numerosa de tags que podem ser utilizadas.

Listas ordenadas e não ordenadas

Voltar ao topo

Um recurso muito utilizado em HTML é a criação de listas. Basicamente há dois tipos de lista que iremos montar, as ordenadas, e as não ordenadas.

Listas Ordenadas

Para montar uma lista ordenada, a tag necessária é <ol>. Entretanto, somente a abertura desta tag não basta para inserir os dados da lista, é necessário também elencar os itens utilizando a tag <li>. Cada abrir e fechar dessa tag será designado como um item diferente da lista. O nome "lista ordenada" se deve ao fato de que ela irá inserir uma numeração em cada item.

Listas Não Ordenadas

Já para montar uma lista não ordenada, a tag necessária é a <ul>. Análogamente é necessário também elencar os itens utilizando a tag <li>. Diferentemente da lista ordenada, a não ordenada utilizará marcadores ao invés de números.

Links

Voltar ao topo

Para o último conteúdo desse curso, iremos abordar a navegação do nosso site. Você deve ter percebido que mesmo essa página simples possui textos sublinhados em cor azulada, os quais, se pressionados, te direcionam para alguma parte da página, para uma outra página do site, ou para fora desse site, direcionando para uma página da internet. Esses são os links.

Para utilizarmos links utilizamos a tag <a>, que designa uma "ancoragem", ou seja, um vínculo entre o elemento ancorado e seu destino, gerando o laço, ou link. Além da tag <a>, é necessária a utilização de um atributo, o "href", onde o programador irá estabelecer o destino do link gerado, podendo ser como os tipos que você verificou nesse site. A linha de código fica dessa forma: <a href="endereço"> texto que irá aparecer no site </a>. O texto escolhido ficará em azul e sublinhado, ao passo que clicar nele irá direcionar o usuário para o endereço estabelecido no "href". Ainda, para que o link seja aberto em uma nova aba, o desenvolvedor deve acrescentar o atributo "target" e o igualar a _blank, dentro da tag "a" de abertura.

Exemplo: <a href="https://www.nttdata.com/global/en/" target="_blank" > Site da NTT Data Global </a>

Observação: Ao não se utilizar o atributo target, o seu link se comporta como se estivesse escrito " "target="_self" ", implicando a abertura do link na mesma aba.

Ainda, se pode adicionar um título que irá aparecer ao usuário quando ele passar o mouse por cima do link ancorado, utilizando o atribuito title. Sua inserção no código é similar ao atributo blank.

Para se navegar em páginas dentro do próprio site, a lógica seguida é a mesma. Contudo, é necessário primeiramente criar uma nova página no VSC. Após ir em "Explorador" e clicar na opção "Novo Arquivo..." insira o nome da página acrescentando .html ao final. A configuração é análoga ao que está sendo feita na página principal, mudando apenas os conteúdos.

Após a criação da nova página, basta colocar o nome dela no campo de endereço do "href". É importante lembrar que é necessário criar um link de volta para a página principal nas páginas auxiliares.

Boas práticas de programação indicam que links para páginas do site sejam abertas na mesma aba, enquanto que links externos sejam abertos em uma nova aba.
